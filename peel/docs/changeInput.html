Change Input

This jqueryUI widget is attached to each input field that has a change. The widget is comprised of an icon positioned next to the input and a lazily created popup that is shown when the icon is clicked. The popup will show the current state of the change (what value it is) and what the other values are (the broker and carrier). The user can select either one. All inputs that don't have an associated change, have a regular jquery listener added so that if the user modifies them, a change will be created and a changeInput widget will be attached. In the image below, you can see a changeInput widget for the Last Name field. It shows which value is currently used with a check icon. (note that it's possible that neither is selected if the user manually changes the input field. in that case obviously neither row is selected.) The user can go to the next/prev change via the arrow buttons. This will close the current popup and scroll the form down to the next change and open its popup.

Events

update

Parameters
  • e: the event
  • id: the id of the change
  • value: the value being set. {index: null/0...n, code:$string, size:'sm|md|lg', text:$string});
example
changeInput.on('changeinputupdate', function (e, id, value) { /****/ })

next

Parameters
  • e: the event
  • id: the id of the change
example
changeInput.on('changeinputnext', function (e, id) { /****/ })

prev

Parameters
  • e: the event
  • id: the id of the change
example
changeInput.on('changeinputprev', function (e, id) { /****/ })
**Note that all events are prefixed by the widget namespace 'changeinput'. e.g. 'update' event requires a 'changeinputupdate' listener.
