Change Panel

This is the panel that shows a list of changes in the form. It will iterate over the changes given (options.changes[]) and use a specified template to render each one.

Features

- each change will have an icon beside it for the appropriate change type [add/modify/delete] - the change will have a toggle button beside it unless it has been overridden by user (in which case, an icon is shown) - the toggle button will change the value from broker to carrier (before to after..whatever you call the two values). - the panel can be minimized down to one row in order to save space. - the up/down arrows will select the prev/next change. - when a change is selected, the associated input in the form will *wink* (animated background change). - a change can be selected by clicking on the row. if you click on the name in the row (e.g. "Company", it will also open up the associated change input dialog). see ChangeInput - the title will show an icon with the # of changes in it. this will update if you make more changes. - the selected change item will be highlighted. - the current value of the field will always be shown as the first thing in the row and the description to the right. e.g. Company FooBar (changed from MarsBar) .|. .|. current description/related text - the changes are listed in order of appearance in the DOM.

Events

select

  • e: the event
  • showPopup: boolean request to show input popup as part of selection process
  • change: the change selected {type:add/modify/delete, id:$string, values:[]}
example
changePanel.on('changepanelselect', function (e, id, showPopup) { if (showPopup) blah... })

resize

  • e: the event
  • resizeType: 'maximize'|'minimize'
example
changePanel.on('changepanelresize', function (e, id) { if(resizeType==='minimize') blah... })

set

  • e: the event
  • id: the id of the change
  • value: the value being set {index:null/0..n, text:$string, code:$string, size:'sm'|'md'|'lg'
example
changePanel.on('changepanelset', function (e, id, value) { /****/ })
**Note that all events are prefixed by the widget namespace 'changepanel'. e.g. 'select' event requires a 'changepanelselect' listener.
